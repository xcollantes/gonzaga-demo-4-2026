4. **Reference [Landing Page Architecture](./landing-page-architecture.md)** for

## Contributing to Documentation

When making significant changes to the application:

1. **Update relevant documentation** to reflect architectural or design changes
2. **Add new documentation files** for new major features or patterns
3. **Update this README** to include new documentation files
4. **Maintain code examples** and keep them synchronized with actual
   implementation

## Documentation Standards

- Use clear, descriptive headings and subheadings
- Include code examples for technical concepts
- Reference specific file paths and line numbers when applicable
- Update external links and citations regularly
- Maintain consistent formatting and structure across all docs
